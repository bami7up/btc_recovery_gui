# Wallet Recovery GUI

Графический инструмент для **восстановления пароля собственного Bitcoin Core `wallet.dat`** с помощью генерации словаря, запуска `btcrecover`, а также дополнительного этапа через `bitcoin2john.py + Hashcat`.

> ⚠️ Используйте только для своих кошельков и только с резервной копией `wallet.dat`.

## 1. Что умеет инструмент

- Собирает пользовательские данные для генерации паролей: слова, фразы, числа, спецсимволы.
- Генерирует словарь (`passwords.txt`) по шаблонам (`{W}`, `{P}`, `{N}`, `{S}`) и мутациям регистра.
- Запускает `btcrecover` по очереди сценариев: plain / typos1 / typos2 / capslock.
- Показывает мониторинг процесса (время, сценарий, активность, скорость).
- Извлекает hash из `wallet.dat` через `bitcoin2john.py`.
- Запускает Hashcat (обычно mode `11300` для Bitcoin Core wallet.dat).
- Проверяет баланс BTC-адресов через Esplora API.

## 2. Требования

- Python 3.8+
- Tkinter (обычно входит в стандартный Python)
- Git (для удобной установки `btcrecover`)
- `btcrecover` (репозиторий)
- John the Ripper (нужен `bitcoin2john.py`)
- Hashcat (для второго этапа перебора по hash)

Рекомендуемая структура каталога:

```text
btc_recovery/
  wallet_recovery_gui_final.py
  btcrecover/
  btcrecover_venv/
  john/
    run/
      bitcoin2john.py
  hashcat/
    hashcat.exe (или бинарник под Linux/macOS)
```

## 3. Быстрый старт

### Шаг 1. Клонируйте/скачайте этот проект

```bash
git clone <URL-ВАШЕГО-РЕПО>
cd btc_recovery_gui
```

### Шаг 2. Запустите GUI

Прямой запуск Python:

```bash
python wallet_recovery_gui_final.py
```

Или через launcher-скрипты (рекомендуется):

Windows:
```bat
start_wallet_gui.bat
```

Linux/macOS:
```bash
./start_wallet_gui.sh
```

Launcher сначала проверяет наличие Python и пытается установить его автоматически (Windows: `winget`, Linux: `apt-get`), после чего запускает GUI.

## 4. Подготовка окружения для btcrecover

### Шаг 1. Скачайте btcrecover

```bash
git clone https://github.com/3rdIteration/btcrecover
```

### Шаг 2. Создайте виртуальное окружение

Linux/macOS:

```bash
python -m venv btcrecover_venv
./btcrecover_venv/bin/python -m pip install --upgrade pip setuptools wheel
./btcrecover_venv/bin/python -m pip install -r btcrecover/requirements.txt
```

Windows:

```powershell
py -3.12 -m venv btcrecover_venv
.\btcrecover_venv\Scripts\python.exe -m pip install --upgrade pip setuptools wheel
.\btcrecover_venv\Scripts\python.exe -m pip install -r btcrecover\requirements.txt
```

### Шаг 3. В GUI укажите пути

Во вкладке **«Настройки»**:
- Папка `btcrecover`
- Папка `btcrecover_venv`
- Рабочая папка

Теперь можно нажать **«⬇ Автоскачать btcrecover»** во вкладке «Настройки», чтобы GUI сам скачал и распаковал `btcrecover` в выбранную папку.
Во вкладке **«Hashcat»** появились кнопки:
- **«⬇ Автоскачать bitcoin2john»** — скачивает John Jumbo и автоматически подставляет путь к `bitcoin2john.py`.
- **«⬇ Автоскачать Hashcat»** — скачивает архив Hashcat (для Windows, требуется распаковка `.7z` через 7-Zip).

Нажмите кнопку проверки/инициализации окружения в GUI.

## 5. Полная пошаговая работа в интерфейсе

## Шаг 1. Вкладка «Данные»

Заполните:
- **Слова `{W}`** — имена, ники, значимые слова.
- **Фразы `{P}`** — памятные выражения.
- **Цифры `{N}`** — годы, даты, коды.
- **Спецсимволы `{S}`** — `! @ # _ -` и т.д.

Нажмите **«Сохранить данные»**.

## Шаг 2. Вкладка «Генерация»

1. Настройте мутации:
   - `word / Word / WORD`
   - `swapcase`
   - `PaRoL` / `pArOl`
   - RU→EN раскладка
   - реверс строк
2. Укажите шаблоны (по умолчанию уже добавлены), например:
   - `{W}{N}`
   - `{P}{S}{N}`
   - `{S}{W}{N}`
3. Укажите выходной файл (обычно `passwords.txt`).
4. Нажмите **«СГЕНЕРИРОВАТЬ СЛОВАРЬ»**.
5. При необходимости используйте оценку размера перед генерацией.

## Шаг 3. Вкладка «btcrecover»

1. Выберите **копию** `wallet.dat`.
2. Выберите словарь `passwords.txt`.
3. Отметьте нужные сценарии:
   - Просто словарь
   - Typos 1
   - Typos 2
   - CapsLock
4. Нажмите **«ЗАПУСТИТЬ btcrecover»**.
5. Следите за логом и блоком мониторинга.

Если пароль найден — сохраните результат безопасно.

## Шаг 4. Вкладка «Hashcat» (опционально)

Используйте, если `btcrecover` не дал результат.

1. Укажите путь к `bitcoin2john.py`.
2. Укажите путь к `hashcat`/`hashcat.exe`.
3. Нажмите извлечение hash из `wallet.dat`.
   - GUI автоматически создаёт безопасную копию `wallet.dat` в `safe_wallet_copies/` и работает с копией, чтобы не трогать исходный файл.
4. Укажите mode (обычно `11300`) и workload.
5. Запустите Hashcat.
6. Анализируйте вывод и результат.

## Шаг 5. Вкладка «Баланс»

- Добавьте BTC-адрес(а), проверьте баланс через Esplora API.
- Можно нажать **«🧩 Извлечь адреса из файла»** и загрузить `.txt/.json/.csv/.log` (например, dump/экспорт), GUI сам найдёт BTC-адреса и добавит их в список.

## 6. Рекомендации по эффективности

- Начинайте с узкого, «человеческого» словаря (реальные варианты).
- Увеличивайте пространство постепенно: сначала простые шаблоны, потом typos и сложные комбинации.
- Контролируйте размер словаря: слишком большие файлы сильно тормозят перебор.
- Храните историю изменений входных данных (что уже проверяли).

---

## 7. Безопасность

- Никому не отправляйте `wallet.dat`, hash и словари.
- Работайте офлайн, если возможно.
- Используйте только копии кошелька.
- Делайте резервные копии перед экспериментами.
- Приложение делает автосохранение настроек и данных (примерно каждые 15 секунд) и сохраняет при закрытии окна, чтобы после перезапуска восстановить введённые значения.
- Если основной файл сохранения повреждён (например, при сбое), при старте приложение пытается восстановиться из резервной копии (`.bak`) или временного файла (`.tmp`).

## 8. Типовые проблемы и решения

### `btcrecover.py` не найден
Проверьте путь к папке `btcrecover` во вкладке «Настройки».

### `requirements.txt` не найден
Проверьте, что репозиторий `btcrecover` корректно клонирован.

### Ошибки модулей при работе `bitcoin2john.py`
Попробуйте установить `bsddb3` в venv:

```bash
python -m pip install bsddb3
```

(или через интерпретатор вашего `btcrecover_venv`).

### Hashcat завершается с ошибкой
- Проверьте правильность mode (`11300` для многих `wallet.dat` Bitcoin Core).
- Проверьте путь к бинарнику Hashcat.
- Проверьте, что hash-файл сформирован корректно.
